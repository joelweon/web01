이벤트 처리 - 이벤트 등록 I

1) onXxx 이름으로 이벤트 리스너(핸들러) 등록하기
   => onXxx 이름은 이벤트 별로 미리 정의되어있다.
   => 이벤트가 발생하면 자바스크립트 엔진은 그 이벤트에 대한 함수가 등록되었는지
      조사하여 등록되어 있을 때 호출한다.
   => 단점: 이벤트 리스너를 오직 한 개만 등록할 수 있다.
   
2) addEventListener(이벤트명, 함수)를 호출하여 이벤트 리스너를 등록하기
   => 장점: 이벤트 리스너를 여러 개 등록할 수 있다.
   => MS IE 브라우저의 경우 버전 9부터 지원한다.
     - 이전 버전은 메서드 명이 다르다. attachEvent()/detachEvent()
     - MS IE 8 이하 버전에서도 동작하게 하려면 조건문으로 검사해서
       추가 코드를 작성해야 한다. (아직 관공서는 IE8인 경우가 많다.)
3) HTML 태그의 속성에 자바스크립트 코드를 직접 넣는 방식
   => 간단히 처리할 때 좋다.
   => 그러나 HTML과 자바스크립트 코드가 뒤섞이기 때문에
      나중에 유지보수 할 때 소스코드 가독성이 떨어진다.
